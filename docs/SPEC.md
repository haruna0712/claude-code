## 0. プロダクト概要

エンジニアによる、エンジニアのための SNS。X（旧 Twitter）をベースに、コードスニペット投稿、技術タグ、Zenn ライク記事機能、5ch ライク掲示板を統合し、エンジニアコミュニティの情報流通と交流を促進する。

### 0.1 ターゲット

- 主要: 日本語話者のソフトウェアエンジニア（Web/モバイル/インフラ/AI 等の現職・学習者）
- 副次: テックライター、技術系ブロガー、技術系学生

### 0.2 差別化ポイント

|                | X                 | Zenn | 5ch | 本SNS                                 |
| -------------- | ----------------- | ---- | --- | ------------------------------------- |
| 短文投稿       | ✅                | ❌   | ❌  | ✅（180字+ Markdown +コードブロック） |
| 技術タグ       | △（ハッシュタグ） | ✅   | ❌  | ✅（最大3個・サジェスト式）           |
| 記事           | ❌                | ✅   | ❌  | ✅（GitHub 連携）                     |
| 掲示板         | ❌                | ❌   | ✅  | ✅（管理者管理＋アカウント名表示）    |
| DM             | ✅                | ❌   | ❌  | ✅（1:1+グループ）                    |
| 自動IT記事配信 | ❌                | ❌   | ❌  | ✅（RSS+AI 要約）                     |

### 0.3 用語

| 用語                  | 定義                                                   |
| --------------------- | ------------------------------------------------------ |
| ツイート（Post）      | 180字以内・Markdown 対応の短文投稿                     |
| リツイート（Repost）  | 他人のツイートをそのままTLに再共有                     |
| 引用ツイート（Quote） | 他人のツイートに自分のコメントを添えて投稿             |
| リプライ（Reply）     | ツイートに対する返信                                   |
| リアクション          | 固定絵文字セットによる反応（X の Like 拡張）           |
| タグ                  | ツイート/記事に最大 3 個まで付与する技術カテゴリ       |
| お気に入りボックス    | 他者のツイートを保存する非公開コレクション             |
| 板（Board）           | 掲示板の最上位カテゴリ。管理者のみ作成・削除可         |
| スレッド（Thread）    | 板内のトピック。誰でも作成可。最大 1000 レス           |
| 記事（Article）       | Markdown 形式の長文。Zenn ライク。公開/限定公開/下書き |
| Bot                   | RSS から自動投稿するシステムアカウント                 |

## 1. 認証・アカウント

### 1.1 認証方式

- **メール + パスワード**（djoser ベース、既存テンプレートを活用）
- **Google OAuth 2.0**（`social-auth-app-django` を追加）
- 2FA は MVP では未実装（将来 TOTP を予定）

### 1.2 サインアップフロー

1. メール・パスワード・表示名・@handle を入力
2. djoser がアクティベーションメール（Mailgun 経由）を送信
3. メール内リンククリックで `is_active=True`
4. 初回ログイン直後にプロフィール初期設定ウィザードへ誘導（職種・スキルタグ・興味関心）

### 1.3 ログイン・トークン

- JWT（`rest_framework_simplejwt`）
- アクセストークン HttpOnly Cookie 保存（既存 `apps.common.cookie_auth` 流用）
- リフレッシュトークン HttpOnly Cookie・SameSite=Lax
- セッション有効期間: アクセス 60 分 / リフレッシュ 14 日

### 1.4 パスワードリセット

- djoser 標準フロー
- メール経由でリセットリンク送信（24h 有効）

### 1.5 アカウント削除

- 設定画面から実行
- 物理削除（GDPR 配慮、ただし監査用最小ログのみ 90 日保持）
- 削除済みユーザーのツイートも連動して物理削除
- リプライ元として参照されている場合は「削除されたユーザー」表示

## 3. ツイート（Post）

### 3.1 仕様

| 項目           | 値                                                                                         |
| -------------- | ------------------------------------------------------------------------------------------ |
| 文字数上限     | **180 文字**（プレミアム会員は 500 文字）                                                  |
| カウント方式   | 表示文字数（Markdown 記号を除外）                                                          |
| Markdown       | フル対応（GFM 準拠：見出し/リスト/リンク/画像/引用/表/打消し/タスクリスト/コードブロック） |
| コードブロック | ` ```lang ... ``` ` 形式、シンタックスハイライト（Prism.js or Shiki）                      |
| 画像添付       | 最大 4 枚 / 1 枚 5 MB / JPG・PNG・WebP・GIF                                                |
| 動画添付       | **なし**（MVP）                                                                            |
| 編集           | **投稿後 30 分以内・最大 5 回・編集履歴を表示**（X 準拠）                                  |
| 削除           | 物理削除。削除後はリプライ元として「削除済みツイートです」表示                             |
| メンション     | `@handle` で通知トリガー                                                                   |
| ハッシュタグ   | **本文内 `#` は廃止**、タグは UI で別途付与                                                |
| タグ           | 最大 3 個（後述）                                                                          |

### 3.2 ツイートタイプ

- `original` ... 通常投稿
- `reply` ... 親ツイートへの返信
- `repost` ... リツイート（参照のみ・本文なし）
- `quote` ... 引用ツイート（参照 + 本文 180 字）
- `bot_news` ... Bot による自動投稿（記事カード付き）
- `from_article` ... 自分の記事公開時の自動連携投稿（記事マーク付き）

### 3.3 リツイート（Repost）仕様

- ボタン押下で即時 RT、再押下で取消
- 元ツイートの `repost_count` をインクリメント
- 自分の TL とフォロワーの TL に元ツイートが表示される
- RT 主のアカウント名と「○○がリポストしました」表示（X 準拠）

### 3.4 引用ツイート（Quote）仕様

- 元ツイートをカード形式で埋め込み、自分のコメント（180字）を上部に
- 元ツイートの `quote_count` をインクリメント
- 通知: 元ツイート主に「引用されました」

### 3.5 OGP カード生成

- 本文中の URL を検出 → 非同期 Celery タスクで OGP メタを取得
- 24h キャッシュ（Redis）
- 1ツイートにつき最初の URL 1件のみカード化

### 3.6 「もっと見る」展開

- X 準拠: 画像・動画・コードブロック含む場合に本文の高さを制限し、「もっと見る」ボタンで全文展開
- テキストのみ 180 字までは常時全文表示（min-height は確保せず短文は短いまま）

### 3.7 文字数カウント仕様（詳細）

````
表示文字数 = レンダリング後のテキスト文字数
　- Markdown 記号（**, _, #, [], () 等）は除外
　- 画像 Markdown は 1 件 = URL 短縮分として 23 文字カウント（X方式）
　- コードブロック内は文字数カウント対象（記号 ``` は除く）
````

## 4. タグ

### 4.1 仕様

- ツイート 1 つに最大 **3 個**
- タグ名: 英数字 + `-` のみ・最大 20 字
- 大文字小文字は区別しない（保存時に小文字化）
- エイリアス管理は行わない（公式表記に統一誘導）

### 4.2 タグ作成

- 入力時に既存タグからインクリメンタルサーチ・サジェスト
- マッチしない場合は新規作成可能
- **新規作成時の編集距離チェック**（表記ゆれ乱立防止）:
  - 既存タグとの Levenshtein 距離 ≤ 2 のタグが存在する場合、「もしかして `<候補>` ですか？」を表示
  - ユーザーが「新規作成」を選べば作成可能、既存タグを選べば差し替え
  - `is_official=True` の公式タグが距離 ≤ 2 に存在する場合は新規作成をブロック、公式タグ利用を強制
- 初期シードタグ（管理者投入）:
  - 言語: `python`, `go`, `rust`, `typescript`, `javascript`, `java`, `kotlin`, `swift`, `c`, `cpp`, `csharp`, `ruby`, `php`, `scala`, `elixir`, `haskell`, `r`, `dart`, `lua`, `perl`
  - フロント: `react`, `nextjs`, `vue`, `nuxt`, `svelte`, `angular`, `tailwindcss`, `vite`
  - バック: `django`, `fastapi`, `flask`, `rails`, `laravel`, `spring-boot`, `nestjs`, `express`, `gin`, `actix`
  - インフラ: `aws`, `gcp`, `azure`, `kubernetes`, `docker`, `terraform`, `ansible`, `nginx`, `linux`
  - DB: `postgresql`, `mysql`, `mongodb`, `redis`, `elasticsearch`, `clickhouse`, `dynamodb`
  - データ/AI: `pytorch`, `tensorflow`, `huggingface`, `langchain`, `claude-code`, `chatgpt`, `gemini`, `ollama`
  - その他: `github`, `gitlab`, `vscode`, `vim`, `neovim`, `emacs`

### 4.3 タグページ

- `/tag/<name>` で公開（未ログイン閲覧可）
- そのタグが付いたツイート・記事を新着順で一覧表示
- フォロー（タグ購読）は MVP では未実装

### 4.4 トレンドタグ

- 過去 24h で付与回数の多いタグ Top 10 を表示
- Celery Beat で 30 分ごとに集計、Redis にキャッシュ

## 5. タイムライン

### 5.1 ホームタイムライン

**アルゴリズム配信**:

- 70%: フォロー中のユーザーのツイート（時系列、新しい順、24h 以内）
- 30%: 全体のリアクション数上位ツイート（過去 24h）
- 重複は除外、同じ著者の連投は最大 3 件まで
- Celery Beat で 5 分ごとにユーザー別 TL を Redis にキャッシュ（ZSET、有効期限 10 分）

### 5.2 タブ構成

- ホーム（アルゴリズム）
- フォロー中（時系列のみ）
- 通知
- DM
- 検索
- 掲示板
- 記事

### 5.3 おすすめユーザー（Who to follow）

- 興味関心タグが登録済 → 同じタグを多く投稿しているユーザー上位
- 興味関心が空 → 自分のリアクション履歴のタグから推定
- リアクション履歴も無い → 全体フォロワー数上位
- バッチ: Celery Beat で 1 時間ごとに更新

### 5.4 ペジネーション

- カーソル方式（`?cursor=<base64-encoded-id-and-timestamp>`）
- 1 ページ 20 件

## 6. リアクション

### 6.1 仕様

- 1 ユーザーが 1 ツイートに付けられるリアクションは **1 種類のみ**（X 式）
- 同じリアクションを再押下で取消
- 別の種類に変更も可能（自動で前のリアクションを取消）

### 6.2 絵文字セット（固定 10 種）

| キー          | 絵文字 | ラベル       |
| ------------- | ------ | ------------ |
| `like`        | ❤️     | いいね       |
| `interesting` | 💡     | 面白い       |
| `learned`     | 📚     | 勉強になった |
| `helpful`     | 🙏     | 助かった     |
| `agree`       | 👍     | わかる       |
| `surprised`   | 😲     | びっくり     |
| `congrats`    | 🎉     | おめでとう   |
| `respect`     | 🫡     | リスペクト   |
| `funny`       | 😂     | 笑った       |
| `code`        | 💻     | コードよき   |

ネガティブ系（Bad、Dislike 等）は意図的に**追加しない**。

## 7. DM（ダイレクトメッセージ）

### 7.1 仕様

- **リアルタイム配信**（Django Channels + WebSocket）
- 1:1 + グループチャット
- グループ最大 20 名
- グループ作成者のみが他メンバーを招待可能
- 招待を受けたユーザーは承認制（拒否も可能）
- 既読表示・タイピング中表示・画像/ファイル送信対応

### 7.2 グループ招待フロー

1. グループ作成者が「招待」から `@handle` を指定
2. 招待された人に通知（アプリ内通知）
3. 受諾 → グループに参加・メッセージ閲覧開始
4. 拒否 → 通知から消去、招待者には特に通知しない（A13 確定）

### 7.3 メッセージ仕様

- 本文: Markdown 対応
- 添付: 画像（JPG/PNG/WebP/GIF, 1ファイル 10MB, 最大 5 ファイル/送信）、汎用ファイル（PDF/ZIP/etc, 最大 25 MB, 1 ファイル/送信）
- メッセージ削除: 自分の送信は物理削除可能、相手側の表示も削除
- メッセージ編集: なし（MVP）

### 7.4 既読・タイピング

- メッセージ既読: ユーザー単位で `last_read_at` を保持し、それ以降のメッセージを未読扱い
- タイピング: WebSocket で `typing` イベントを送出、3 秒後自動消去

### 7.5 通知

- 新着メッセージ: アプリ内通知（メール送信なし）
- グループ招待: アプリ内通知

## 8. 通知

### 8.1 通知種別

| 種別              | トリガー                                    | デフォルト |
| ----------------- | ------------------------------------------- | ---------- |
| `like`            | 自分のツイートにリアクション                | ON         |
| `repost`          | 自分のツイートを RT                         | ON         |
| `quote`           | 自分のツイートを引用                        | ON         |
| `reply`           | 自分のツイートに返信                        | ON         |
| `mention`         | ツイート/記事/掲示板で `@handle` メンション | ON         |
| `dm_message`      | 新着 DM                                     | ON         |
| `dm_invite`       | グループ招待                                | ON         |
| `follow`          | フォローされた                              | ON         |
| `article_comment` | 自分の記事にコメント                        | ON         |
| `article_like`    | 自分の記事にいいね                          | ON         |

### 8.2 ユーザー設定

- 設定画面で**種別ごとに ON/OFF** 切替可能
- 既定値は全 ON

### 8.3 通知配信

- 配信先: **アプリ内通知のみ**
- メール通知は送信しない（メールはサインアップ・パスワードリセット・運営お知らせのみ）
- Web Push は MVP では未実装

### 8.4 通知 UI

- ヘッダーのベルアイコンに未読数バッジ
- 通知一覧ページで既読/未読フィルタ
- WebSocket でリアルタイム更新

- ツイート本文

### 10.2 検索インフラ

- **Meilisearch**（Docker / ECS Fargate サイドカー）

## 11. 掲示板

### 11.1 構造

```
板（Board）           ← 管理者のみ作成・削除
  └ スレッド（Thread）  ← ユーザー作成可、最大1000レス
      └ レス（Post）    ← ログインユーザーのみ投稿
```

### 11.2 仕様

- **未ログインで閲覧可能**（SEO・集客のため）
- 投稿はログイン必須（未ログイン時は投稿欄に「ログインが必要です」と CTA 表示）
- 匿名投稿不可、`@handle` とアイコンを表示
- 1 スレッド最大 1000 レス → 990 レス目で「新スレ作成を促す」バナー、1000 達成で投稿停止＋「次スレへ」リンク表示
- スレッド削除は**管理者のみ**
- スレッド作成者は自分のスレを削除できない（モデレーションのため）
- レス削除は投稿者本人 + 管理者
- メンションは `@handle`（5ch 風 `>>1` は廃止）
- レス本文: Markdown / コードブロック / 画像貼付け OK
- 画像: 1 レスにつき最大 4 枚・各 5MB

### 11.3 板の管理

- Django admin で作成・編集・削除
- 板ごとに名前・説明・並び順・アイコン色

### 11.4 検索

- 掲示板内検索は MVP では未実装（ツイート/記事検索とは分離）
- 将来 Meilisearch インデックスに追加予定

## 12. 記事（Zenn ライク）

### 12.1 仕様

- アプリ内 **Markdown エディタ**（プレビュー付き、画像ドラッグ&ドロップ）
- 任意で **GitHub 連携**（**MVP は片方向: アプリ内編集 → GitHub push のみ**、pull は Phase 11 以降）
- 画像: **S3 保存**（記事ごとに `articles/<article-id>/<filename>`）
- ステータス: **下書き / 公開** の 2 段階（MVP から限定公開は除外）
- リアクション: いいね（1 ユーザー 1 件）
- コメント: ログイン必須、Markdown 対応、ネスト 1 段階まで
- タグ: ツイートと共通の Tag マスタ・最大 5 個

### 12.2 公開記事

- URL: `/articles/<slug>` または `/articles/<id>`
- 未ログインで閲覧可能
- OGP 自動生成（タイトル + 著者 + サムネ）
- JSON-LD（`Article` schema）出力で SEO 強化

### 12.3 GitHub 連携（MVP: 片方向 push のみ）

- ユーザーが GitHub OAuth でリポジトリへの write 権限を許可（`public_repo` スコープ）
- リポジトリ構成（Zenn 方式）:
  ```
  articles/
    my-first-article.md  ← Front Matter にメタ
  images/
    my-first-article/
      hero.png
  ```
- Front Matter 例:
  ```yaml
  ---
  title: "Django + Next.js で作る SNS"
  emoji: "🐍"
  type: "tech"
  topics: ["django", "nextjs", "aws"]
  published: true
  ---
  ```
- **MVP の挙動**:
  - アプリ内で記事作成・編集・公開 → GitHub リポジトリに push（commit message: `docs: <title> を更新`）
  - 画像も `images/<slug>/` 配下に push
  - GitHub 側で直接編集された変更の取り込み（pull / Webhook 受信）は **Phase 11 以降**
  - 双方向同期ではないため、ユーザーには「記事編集はアプリ内で行ってください（GitHub は公開バックアップ用途）」と明示
- OAuth トークンは DB に暗号化保存（`cryptography.fernet` + AWS KMS 由来鍵）

### 12.4 記事公開時の自動ツイート

- デフォルト ON（設定で OFF 可）
- 自動ツイート内容: `タイトル + 冒頭 80 字 + 記事リンク`
- ツイートに**記事マーク（📄 文書アイコン、Lucide `FileText`）**を著者名右に表示
- ツイートクリックで記事ページへ遷移

## 13. 自動投稿 Bot

### 13.1 ソース

| ソース                   | 種別             | Bot アカウント |
| ------------------------ | ---------------- | -------------- |
| ITmedia NEWS             | 国内             | `@itmedia_bot` |
| Hacker News (Front Page) | 海外（翻訳対象） | `@hn_bot`      |

将来的に Publickey, gihyo, Dev.to, TechCrunch 等を追加予定。

### 13.2 処理フロー

1. **RSS 取得**（Celery Beat で 30 分ごと）
2. 既出記事は重複排除（`source_url` の SHA256 を Redis Set で保持・30 日 TTL）
3. **Claude API（記事用 AI）または ChatGPT API（要約用）** に投入:
   - 海外: 日本語翻訳 + 要約 + 感想 + 関連タグ 3 個
   - 国内: 要約 + 感想 + 関連タグ 3 個
4. ChatGPT API（`gpt-4o-mini` 想定）でコスト最適化
5. ツイート投稿（180 字以内に収める）
6. OGP カードとして元記事リンクを添付

### 13.3 投稿フォーマット

```
[感想・コメント 60〜100字]

📄 [記事タイトル（最大 50 字、超過時 …）]
[OGP カード]
```

### 13.4 著作権配慮

- **元文の引用は最小限**（タイトル + 60〜120字の感想中心）
- 必ず元記事へのリンクを付与
- 翻訳記事は「[翻訳・要約] [元タイトル]」と明記
- 画像は元記事 OGP のみ（再アップロードしない）

### 13.5 投稿頻度上限

- 1 ソースあたり 1 時間 1 件
- 1 日最大 24 件 / Bot
- 同一記事の重複投稿禁止

### 13.6 Bot アカウント表示

- プロフィールに「🤖 Bot アカウント」明示
- 一般ユーザーはフォロー可、リプライ可だが Bot は反応しない

## 14. モデレーション

### 14.1 管理者ロール

- **admin のみ**（単一ロール）
- Django admin で全管理操作

### 14.2 ブロック

- 相互非表示（プロフィール・ツイート・DM 不可）
- ブロック解除可能

### 14.3 ミュート

- 相手は気付かない
- 自分の TL ・通知 から相手の投稿を非表示
- ミュート解除可能

### 14.4 通報（Report）

- 対象: ツイート / 記事 / DM / 掲示板レス / ユーザーアカウント
- 通報理由: スパム / 誹謗中傷 / 著作権侵害 / 不適切コンテンツ / その他（自由記述）
- 通報は管理者ダッシュボードに集約、対応済みフラグで管理

### 14.5 自動スパム検知（階層化）

1 時間ごとの Celery Beat ジョブで検知:

| 閾値                | アクション                                                         |
| ------------------- | ------------------------------------------------------------------ |
| 100 ツイート/日 超  | 当該ユーザーの画面上部に「投稿ペースが速い状態です」注意バナー表示 |
| 500 ツイート/日 超  | DRF Throttle で自動レート制限（1 分 5 ツイートまで）               |
| 1000 ツイート/日 超 | 管理者にメール通知（Mailgun）、管理画面で手動対応                  |

自動 BAN は行わず、最終判断は管理者が担う。

## 15. プレミアム機能（Stripe）

### 15.1 プラン

| プラン | 料金                          |
| ------ | ----------------------------- |
| 月額   | ¥500                          |
| 年額   | ¥5,000（年払い 2 ヶ月分お得） |

### 15.2 特典

- **記事下書き AI 生成**（Claude API、月 30 回まで）
  - プロンプト → 記事タイトル + 構成 + 本文ドラフト生成
  - エディタに直接挿入
- **プロフィールバッジ**（Premium マーク表示）
- **ツイート文字数拡張**（180 → 500 字）
- **将来的な広告非表示**（広告導入時）

### 15.3 決済フロー

- Stripe Checkout（Hosted ページ）
- Webhook で `subscription.created/updated/deleted` を受信
- ユーザーモデルに `is_premium`, `premium_expires_at` を保持
- 年払い: 1 年後の月初に自動更新
- 解約: 即日無効化ではなく**期間終了まで利用可**

### 15.4 失効時の挙動

- 500 字ツイートは保存済みのものはそのまま表示・編集は可
- バッジは即時非表示
- 記事 AI 生成は不可

## 17. 非機能要件

### 17.1 パフォーマンス

- LCP < 2.5s（ホーム・記事・プロフィール）
- API レスポンス p95 < 500ms
- WebSocket 接続維持 1000 同時 / Channels インスタンス

### 17.2 アクセシビリティ

- WCAG 2.2 AA を目標（詳細は別途 a11y-architect 設計）
- キーボード操作・スクリーンリーダー対応
- カラーコントラスト 4.5:1 以上

### 17.3 セキュリティ

- HTTPS 強制（HSTS）
- CSP（nonce ベース）
- XSS 対策: ユーザー Markdown は `bleach` (サーバ側) + DOMPurify (クライアント側) で二重サニタイズ
- CSRF: djoser + Cookie 認証で対応
- 通報・ブロック機能、レート制限（DRF Throttle）
- 秘匿情報: AWS Secrets Manager（Django SECRET_KEY, DB password, API keys）
- **Webhook エンドポイントは専用サブドメイン (`webhook.*`) で ALB 直受信**（CloudFront 非経由で署名検証の破損を防止、Stripe/GitHub 両方）
- Webhook 受信時は公式 IP レンジのみ許可 + HMAC 署名検証必須

### 17.4 監査・ログ

- 退会時データ削除（90 日後完全削除、それまで監査用）
- ログイン履歴・パスワード変更履歴を 1 年保持
- Sentry でエラートラッキング（フロント・バック両方）

### 17.5 i18n / Timezone

- UI: 日本語のみ（MVP）
- タイムゾーン: JST 固定
- 英語 UI は Phase 7 以降
